70 /******************************************************************************
71  * NOTE 1: This project provides two demo applications. A simple blinky style
72  * project, and a more comprehensive test and demo application. The
73  * mainCREATE_SIMPLE_BLINKY_DEMO_ONLY setting in main.c is used to select
74  * between the two. See the notes on using mainCREATE_SIMPLE_BLINKY_DEMO_ONLY
75  * in main.c. This file implements the simply blinky style version.
76  *
77  * NOTE 2: This file only contains the source code that is specific to the
78  * full demo. Generic functions, such FreeRTOS hook functions, and functions
79  * required to configure the hardware, are defined in main.c.
80  ******************************************************************************
81  *
82  * main_blinky() creates one queue, and two tasks. It then starts the
83  * scheduler.
84  *
85  * The Queue Send Task:
86  * The queue send task is implemented by the prvQueueSendTask() function in
87  * this file. prvQueueSendTask() sits in a loop that causes it to repeatedly
88  * block for 200 milliseconds, before sending the value 100 to the queue that
89  * was created within main_blinky(). Once the value is sent, the task loops
90  * back around to block for another 200 milliseconds.
91  *
92  * The Queue Receive Task:
93  * The queue receive task is implemented by the prvQueueReceiveTask() function
94  * in this file. prvQueueReceiveTask() sits in a loop where it repeatedly
95  * blocks on attempts to read data from the queue that was created within
96  * main_blinky(). When data is received, the task checks the value of the
97  * data, and if the value equals the expected 100, toggles the LED. The 'block
98  * time' parameter passed to the queue receive function specifies that the
99  * task should be held in the Blocked state indefinitely to wait for data to
100  * be available on the queue. The queue receive task will only leave the
101  * Blocked state when the queue send task writes to the queue. As the queue
102  * send task writes to the queue every 200 milliseconds, the queue receive
103  * task leaves the Blocked state every 200 milliseconds, and therefore toggles
104  * the LED every 200 milliseconds.
105  */
